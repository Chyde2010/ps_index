# Live Track Record -- Bug Report
## Month affected: May 2026 (2026-05)
## Date identified: May 2026
## Date fixed: May 2026 (before June 1st automated run)

---

## Bug description

The May 2026 automated signal run (fired May 1st 2026 via
GitHub Actions) produced own_z=0.000 for all nine tickers
in the universe. This caused incorrect portfolio weights
and missed the BABA High Conviction signal that should have
fired this month.

---

## Root cause

Two path resolution bugs in generate_signal.py:

1. OUTPUT_DIR, HISTORY_FILE, WEIGHTS_FILE and SERIES_DIR
   were defined as relative paths ('live_track_record/...').
   When GitHub Actions runs the script from the repo root,
   the working directory causes these paths to resolve
   incorrectly.

2. BABA and MNDY ps_series seed files were missing entirely.
   These two tickers were added to the universe in May 2026
   but their historical ps_series CSV files (required for
   own-history z-score calculation) had not been committed
   to the repo before the May 1st run.

Combined effect: load_own_history() returned empty DataFrames
for all tickers. own_history_zscore() returned 0.0 when
history length < 3. All own_z scores defaulted to 0.000.

---

for May 2026. The corrected signal would have added BABA
as a long HC position alongside the DDOG short HC position.

---

## Portfolio impact

The May 2026 portfolio positions used equal-weight fallback
for the long book (no HC long signals) instead of
overweighting BABA which should have been in HC.

Observed long book weights (May 2026):
  MSFT  17.4%  AMZN  17.4%  CRM  26.1%
  SNOW  17.4%  BABA  21.7%

Corrected long book weights would have been:
  BABA  overweighted (HC signal active, own_z ~2.39)
  Remaining four tickers underweighted proportionally

The DDOG short HC signal was unaffected -- it was correctly
identified and weighted in both the buggy and corrected runs.

---

## Decision: accept May 2026 as-is

In accordance with the pre-commitment discipline, the May
2026 signal is accepted as generated by the automated run.
Retrospective adjustments to signal outputs -- even to
correct a genuine technical error -- would undermine the
integrity of the live track record.

The bug is documented here with full transparency. The
corrected signal is noted for reference. The track record
will show May 2026 as a month where BABA's HC signal was
missed due to a software bug.

This decision is consistent with the methodology: the
pre-commitment timestamps are inviolable, and the signal
outputs of each monthly run are accepted as they stand.

---

## Fix applied

Three fixes committed to generate_signal.py before the
June 1st automated run:

1. OUTPUT_DIR changed from relative string to:
   os.path.dirname(os.path.abspath(__file__))
   All derived paths (HISTORY_FILE, WEIGHTS_FILE,
   SERIES_DIR) now resolve correctly regardless of
   working directory.

2. BABA ps_series seed file committed:
   live_track_record/ps_series/BABA_ps_series.csv
   123 months of historical data seeded from pipeline
   output (Jan 2019 to Mar 2026).

3. MNDY ps_series seed file committed:
   live_track_record/ps_series/MNDY_ps_series.csv
   58 months of historical data seeded from pipeline
   output (Jun 2021 to Mar 2026).

All fixes verified via dry run in Colab on May 2026.
June 1st automated run expected to produce correct
own_z scores for all nine tickers.
